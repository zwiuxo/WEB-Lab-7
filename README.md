# Лабораторная работа №7 — Асинхронная обработка данных (Очереди сообщений)

**👩‍💻 Автор**
- **ФИО:** Лащев Владислав
- **Группа:** ПМ-ИП1
- **Вариант:** 16 (Четный — RabbitMQ)

**📌 Описание задания**
- Научиться работать с брокерами сообщений (RabbitMQ) через PHP.
- Реализовать асинхронную обработку заказов такси: сайт мгновенно принимает заказ, а фоновый процесс (Worker) его обрабатывает.
- Разделить логику на Producer (отправитель) и Consumer (обработчик).

**🚨 Штрафное задание (Выполнено)**
- **Две очереди:** Реализована основная очередь `taxi_main` и очередь для ошибок `taxi_errors`.
- **Обработка ошибок:** Если воркер не может обработать сообщение , оно не пропадает, а автоматически переносится во вторую очередь.
- **Статистика:** На главной странице (`index.php`) выводится количество сообщений в обеих очередях в реальном времени.

**⚙️ Как запустить проект**
1. Собрать контейнеры (может потребоваться VPN для скачивания образов):
```bash
docker-compose up -d --build
```
2. Установить библиотеки (в папке проекта):
```bash
docker run --rm -v \${PWD}/www:/app composer install
```
3. Запустить фоновый обработчик (Worker) в отдельном терминале:
```bash
docker exec -it lab7_php php worker.php
```
4. Ссылки:
- **Сайт:** [http://localhost:8080](http://localhost:8080)
- **Панель RabbitMQ:** [http://localhost:15672](http://localhost:15672) (логин/пароль: guest/guest)

**📂 Содержимое проекта**
- `QueueManager.php` — класс для работы с RabbitMQ (отправка, чтение, статистика).
- `send.php` — скрипт-отправитель (Producer).
- `worker.php` — скрипт-обработчик (Consumer) с логикой перехвата ошибок.
- `index.php` — форма заказа и мониторинг состояния очередей.
